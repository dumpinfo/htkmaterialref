5.8.7 AIFF File Format

  The AIFF format was defined by Apple Computer for storing monoaural and multichannel sampled sounds. An AIFF file consists of a number of chunks. A Common chunk contains the fundamental parameters of the sound (sample rate, number of channels, etc) and a Sound Data chunk contains sampled audio data. HTK only partially supports AIFF since some of the information in it is stored as floating point numbers. In particular, the sample rate is stored in this form and to avoid portability problems, HTK ignores the given sample rate and assumes that it is 16kHz. If this default rate is incorrect, then the true sample period should be specified by setting the SOURCERATE configuration parameter. Full details of the AIFF format is available from Apple Developer Technical Support.
